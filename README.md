# 搬瓦工 DC2 机房到底怎么样？线路、延迟、丢包率实测，以及该选哪个套餐才能用上它

朋友问我搬瓦工 DC2 机房还值不值得用，我想了想，发现这个问题其实分两层：DC2 本身怎么样是一回事，你能不能买到、该怎么买又是另一回事。两层都说清楚，才算回答了这个问题。

**搬瓦工 DC2 QNET 机房**，官方编号 USCA_2，是搬瓦工早期上线的洛杉矶机房之一，物理位置在 QuadraNet（简称 QNET）的数据中心里。搬瓦工把它定位为入门级洛杉矶线路，主要覆盖电信用户，联通和移动的表现要打些折扣。

---

## DC2 QNET 机房线路是什么情况

先说结论，DC2 走的是 CN2 GT 线路，但有个关键细节：这不是搬瓦工自己买断的 CN2 GT，而是 QuadraNet 机房层面自己接入的。换句话说，CN2 GT 不保证——理论上机房随时可能调整，搬瓦工也不承诺。

和它形成对比的是 DC3 CN2，那才是搬瓦工自己向 QuadraNet 购买的 CN2 GT 带宽，有保证，效果也更稳定。DC2 和 DC3 本质上都在 QuadraNet 机房里，区别就在这一点上。

回程路由情况：

- **电信**：接入 CN2 GT 回程，延迟大约在 170ms 上下
- **联通**：直连，但走普通线路，延迟偏高
- **移动**：直连普通线路，三者里表现最差

晚高峰期的丢包率是 DC2 最大的短板。部分节点的丢包能跑到 6%–8%，实际用起来感受得到——视频卡顿、SSH 操作延迟、下载速度忽高忽低都是典型症状。电信用户白天体验还行，到了晚上就要靠运气了。

如果你是联通或移动用户，DC2 基本不在推荐列表里。

## DC2 有没有专属套餐

没有。准确说是"早就没了"。

DC2 QNET 曾经有过专门针对这个机房的促销套餐，但那是很多年前的事情了，现在官网上找不到任何以 DC2 QNET 为主力机房的独立套餐。想用 DC2 只有一种办法：买了 KVM 套餐或者 CN2 GIA-E 套餐之后，在 KiwiVM 后台手动迁移过去。

所以问题就变成了：应该买哪个套餐，才能换到 DC2？

## 买了之后怎么切换到 DC2

这一步很多人卡在这里，其实操作不难：

1. 购买完成后，登录搬瓦工官网进入 KiwiVM 控制面板
2. 左侧菜单找到 "Migrate to another DC"（迁移到其他机房）
3. 从列表里找到 "US – Los Angeles DC2 QNET (USCA_2)"
4. 点击确认迁移，等几分钟完成

注意：迁移会重启 VPS，原有数据不会丢失，IP 地址会变更。如果新机房用起来不满意，可以再迁回来，KVM 套餐和 CN2 GIA-E 套餐都支持免费迁移。

## DC2 适合谁？不适合谁

说实话，DC2 QNET 在搬瓦工的机房体系里算不上什么明星机房。用它的理由只有一个：你已经买了 KVM 或 CN2 GIA-E 套餐，顺手测试一下，觉得适合自己再留着用。

**适合这类场景**：
- 电信用户，对延迟要求不高，跑个建站、跑个脚本就够了
- 想在多个机房之间对比测试，DC2 是选项之一
- 预算有限，KVM $49.99/年的前提下能接受线路质量的折扣

**直接排除的情况**：
- 联通或移动用户，DC2 对这两家的优化几乎没有
- 对晚高峰稳定性有要求，丢包率容忍度低
- 需要稳定跑代理、流媒体解锁，找 DC6 或 DC9 效果好很多

问题是，如果只是要找个便宜用的入门机器，KVM 套餐买下来之后完全可以先试 DC8 ZNET 或者其他机房，觉得 DC2 有优势再切过去。不需要为了 DC2 专门买什么。

## 一些常见问题

**Q：DC2 QNET 和 DC2 AO 是同一个机房吗？**

不是。DC2 QNET 用的是 QuadraNet 机房，DC2 AO 后来换到了 Coresite 机房，两个是完全不同的物理地址。KVM 套餐里现在看到的 "DC2" 一般指 DC2 AO，而不是 DC2 QNET。

**Q：DC2 能解锁 Netflix 吗？**

基本不行。搬瓦工洛杉矶机房的 IP 大多已经被 Netflix 识别并屏蔽，DC2 也不例外。解锁流媒体不是 DC2 的强项。

**Q：DC2 的测试 IP 是什么？**

官方测试 IP 为 65.49.198.195，可以用这个地址提前 ping 一下看看你的延迟情况再决定要不要切过去。

**Q：KVM 套餐买了之后一定能切到 DC2 吗？**

不保证。如果 DC2 当时满载或者维护中，后台列表可能不显示该机房。切换前先在 KiwiVM 确认可用机房列表。

**Q：搬瓦工支持支付宝吗？**

支持。支付宝、PayPal、信用卡都能用，国内用户直接支付宝付款即可，不需要任何其他操作。
